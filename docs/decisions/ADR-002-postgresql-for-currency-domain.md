# ADR-002: Использовать PostgreSQL для валютного домена

## Status

Accepted

## Date

2026-04-28

## Context

Валютному сервису нужно хранить справочник валют, источники курсов, исторические значения, интервалы актуальности и правила выбора источников. Модель связная и требует ограничений целостности.

QUIK SQL Server остается источником данных для части контура Portfolio Lens, но не должен становиться хранилищем валютного домена.

## Decision

Использовать PostgreSQL как основное хранилище `portfolio-lens-currency`.

## Alternatives Considered

### SQL Server QUIK

Плюсы:
- уже есть в текущем контуре;
- может содержать часть котировок и курсов.

Минусы:
- привязывает валютный сервис к QUIK;
- не подходит как независимое доменное хранилище;
- усложняет развитие источников вне QUIK.

Rejected: хранилище валютного домена не должно зависеть от broker integration.

### Встроенное хранилище или файлы

Плюсы:
- быстро для прототипа;
- меньше инфраструктуры.

Минусы:
- слабая модель историчности;
- сложнее обеспечить ограничения и выборки по датам;
- плохо масштабируется на несколько источников.

Rejected: подойдет для тестов или временного импорта, но не для целевой модели.

## Consequences

- Сервис получает собственную схему и миграции.
- Доменная модель может развиваться независимо от QUIK и брокеров.
- Нужны отдельные миграции, backup и правила подключения к PostgreSQL.
