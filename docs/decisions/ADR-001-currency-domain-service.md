# ADR-001: Выделить валютный домен в отдельный сервис

## Status

Accepted

## Date

2026-04-28

## Context

Валюты и FX-курсы нужны нескольким доменам Portfolio Lens: портфелю, брокерским интеграциям, QUIK-контуру и будущим источникам котировок.

Если оставить правила валют внутри одного брокерского или QUIK-сервиса, остальные домены будут зависеть от чужой модели данных и от SQL Server QUIK. Это плохо подходит для валютного справочника, кросс-курсов и правил выбора источников.

## Decision

Развивать `portfolio-lens-currency` как владельца валютного домена.

Репозиторий сначала дает общие Go-пакеты для базовых валютных типов, затем становится самостоятельным сервисом с собственной моделью данных, API и жизненным циклом.

## Alternatives Considered

### Оставить валюты в каждом сервисе

Плюсы:
- проще начать;
- меньше межсервисных вызовов.

Минусы:
- дублирование правил;
- разные ответы на один и тот же вопрос о курсе;
- сложнее менять приоритеты источников.

Rejected: валюты используются несколькими доменами, поэтому локальные копии быстро расходятся - это уже было заметно на начальном этапе.

### Оставить валюты в QUIK-контуре

Плюсы:
- QUIK уже дает часть курсов и котировок;
- меньше новой инфраструктуры.

Минусы:
- зависимость от QUIK SQL Server;
- плохо подходит для ЦБ РФ, MOEX и брокерских источников;
- валютный домен становится частью broker integration.

Rejected: валютный домен должен быть независим от конкретного источника.

## Consequences

- Остальные домены получают единый источник истины по валютам и FX.
- Появляется отдельный API и отдельная ответственность за доступность валютных данных.
- На раннем этапе нужен аккуратный минимальный контракт, чтобы не построить сервис вокруг первого адаптера.
