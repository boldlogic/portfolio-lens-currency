# portfolio-lens-currency

Владелец валютного домена в Portfolio Lens.

Репозиторий развивается в два шага: сначала как источник общих Go-пакетов для работы с валютами, затем как отдельный сервис для хранения валютного домена, FX-курсов и правил выбора источников.

## Ответственность

`portfolio-lens-currency` отвечает за:

- справочник валют;
- хранение и выдачу FX-курсов;
- агрегацию курсов из внешних источников;
- правила выбора источника курса;
- конвертацию валют;
- кросс-курсы;
- историческую актуальность курсов.

Сервис не отвечает за брокерские интеграции, агрегацию портфеля и рыночные котировки вне валютного домена.

## Граница контекста

```text
portfolio-lens-currency
  != broker integration
  != portfolio aggregation
  != market quotes

  = currency & FX domain
```

## Источники данных

Сервис не зависит напрямую от брокеров и QUIK SQL Server. Внешние данные должны приходить через адаптеры:

- `quik-fx-adapter`;
- `cbr-adapter`;
- `moex-fx-adapter`;
- `tinkoff-fx-adapter`.

Адаптеры поставляют факты о курсах. Валютный сервис нормализует их, хранит и применяет доменные правила выбора курса.

## Структура

- `pkg/currencies` - общий пакет для сервисов Portfolio Lens: ISO 4217, коды валют и валютные пары.
- `internal/domain` - доменная модель будущего сервиса.
- `internal/service` - сценарии работы с валютами, курсами и конвертацией.
- `internal/repository` - интерфейсы и реализации доступа к хранилищу.
- `internal/transport/http` - HTTP API сервиса.
- `migrations` - PostgreSQL DDL для валютного домена.

## Основные сценарии

- получить metadata по валюте;
- получить текущий курс валютной пары;
- получить исторический курс на дату;
- конвертировать сумму из одной валюты в другую;
- принять курс от внешнего адаптера.

## Хранилище

Целевое хранилище сервиса - PostgreSQL.

Основные сущности:

- `currencies`;
- `fx_sources`;
- `fx_rates`;
- `source_priority_rules`.

## API

Планируемая первая версия HTTP API:

- `GET /api/v1/currencies/{code}`;
- `GET /api/v1/fx/rates/current?base=USD&quote=RUB`;
- `GET /api/v1/fx/rates?base=USD&quote=RUB&at=2026-04-28`;
- `POST /api/v1/fx/rates`;
- `POST /api/v1/fx/convert`.

## Порядок развития

1. Стабилизировать общий пакет `pkg/currencies`.
2. Выделить доменную модель и сервисные сценарии без привязки к БД.
3. Добавить PostgreSQL-модель и миграции.
4. Добавить HTTP API для чтения курсов, конвертации и приема фактов от адаптеров.
5. Подключать адаптеры источников по мере готовности доменных правил.
